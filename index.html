Visual Novel: Sins and Misdeeds

Experience Studios' Visual Novel: Sins and Misdeeds

The creation of this visual novel involved the following components:

  • Implementing Design
  • Documentation
  • Development
  • Testing
  • Q/A
  • Deployment

Experience Studios' visual novel, 'Sins and Misdeeds', began as an idea for a video game, but the limitations imposed by a pandemic, deadlines, financial resources, available equipment, and staff, made it necessary to downsize my original concept from a video game to a visual novel.

Implementing Design

After deciding to produce a visual novel that kept the general concept of the video game, I selected my team.

My team consisted on myself and my father. Together we transposed the main ideas of the proposed video game to the visual novel. I created a general outline of how the story would flow and supplied the dialog. Working as a team, my father and I matched pictures to dialog.

Documentation

My father and I selected and modified pictures, mainly covers, from various public domain comic books in his collection. The music used in this production is also in the public domain. Artists were credited when known.

Development

We selected Ren'py as our development engine. We added pictures and dialog as outlined in the examples and tutorials for the developmental engine. Neither my father nor I have ever written code for Python, so we learned as we went along.

Testing

Testing of the game was an ongoing process that involved the entire team. As soon as a few lines of code were written, we would run it, checking for errors in coding. Many times, the code did not execute as expected or did not execute at all. Usually the problem was easy to find, a missing quotation mark or a semicolon instead of a colon. Once the code executed properly, we check the dialog for spelling, errors in grammar and consistency of character names.

Q/A

In Q/A also known as Quality Assurance, was my job my assignment for the team. My tasks included the following things:

  • Test the visual novel game
  • Check the visual novel game for the bugs and glitches not found during the initial development.
  • Make check list of bugs and glitches.
  • Submit check list to them to make corrections.
  • Using check list as guide, check off the items on list that have been if corrected and verify that the glitches or bugs are no longer in the game, if not, leave unchecked.
  • Document what was checked off and record any new bugs or glitches found in the visual novel this time.

Deployment

The Uploading and Publishing the visual novel required my completion of the following steps:

  • Create an account with Itchio.
  • Create the page within the Itchio site where I can upload the visual novel.
  • Upload the visual novel to the site.
  • Post a description of the visual novel on my Itchio page
